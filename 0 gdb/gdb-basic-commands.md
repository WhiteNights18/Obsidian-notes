## 一、启动与退出

| 命令 | 示例 | 作用 |
|------|------|------|
| `gdb <程序>` | `gdb ./test.out` | 启动 GDB 并加载程序 |
| `gdb <程序> <core文件>` | `gdb ./test.out core.1234` | 调试 core dump 文件 |
| `run` 或 `r` | `r` | 运行程序 |
| `quit` 或 `q` | `q` | 退出 GDB |

## 二、断点相关

| 命令 | 示例 | 作用 |
|------|------|------|
| `break <行号>` | `b 12` | 在指定行设置断点 |
| `break <函数名>` | `b main` | 在函数入口设置断点 |
| `break <文件名>:<行号>` | `b test.c:10` | 在指定文件的行设置断点 |
| `info break` | `i b` | 查看所有断点 |
| `delete <断点号>` | `d 1` | 删除指定断点 |
| `disable <断点号>` | `disable 1` | 临时禁用断点 |
| `enable <断点号>` | `enable 1` | 重新启用断点 |

## 三、执行控制

| 命令 | 作用 |
|------|------|
| `next` 或 `n` | 执行下一行（不进入函数） |
| `step` 或 `s` | 执行下一行（进入函数） |
| `continue` 或 `c` | 继续运行直到下一个断点 |
| `finish` | 运行直到当前函数返回 |
| `until <行号>` | 运行到指定行 |
| `kill` | 终止当前调试的程序 |

## 四、查看代码

| 命令 | 示例 | 作用 |
|------|------|------|
| `list` 或 `l` | `l` | 显示当前行附近的源代码 |
| `list <行号>` | `l 20` | 显示指定行附近的代码 |
| `list <函数名>` | `l main` | 显示指定函数的代码 |
| `disassemble` | `disas` | 显示汇编代码 |

## 五、查看变量和内存

| 命令 | 示例 | 作用 |
|------|------|------|
| `print <变量>` | `p a` | 打印变量的值 |
| `print <表达式>` | `p a+b` | 计算并打印表达式 |
| `print /x <变量>` | `p/x a` | 以十六进制打印 |
| `print /d <变量>` | `p/d a` | 以十进制打印 |
| `print /t <变量>` | `p/t a` | 以二进制打印 |
| `display <变量>` | `disp a` | 每次暂停时自动显示变量 |
| `info locals` | `i locals` | 显示当前函数的局部变量 |
| `info args` | `i args` | 显示当前函数的参数 |

## 六、堆栈相关

| 命令                      | 作用           |
| ----------------------- | ------------ |
| `backtrace` 或 `bt`      | 查看函数调用堆栈     |
| `frame <编号>` 或 `f <编号>` | 切换到指定堆栈帧     |
| `up`                    | 向上移动一层堆栈     |
| `down`                  | 向下移动一层堆栈     |
| `info frame`            | 查看当前堆栈帧的详细信息 |

## 七、线程相关

| 命令                    | 作用        |
| --------------------- | --------- |
| `info threads`        | 查看所有线程    |
| `thread <编号>`         | 切换到指定线程   |
| `thread apply all bt` | 显示所有线程的堆栈 |

## 八、实用技巧

| 命令                 | 事例             | 作用          |
| ------------------ | -------------- | ----------- |
| `set var <变量>=<值>` | `set var a=10` | 在调试时修改变量的值  |
| `call <函数>`        | `call test()`  | 调用函数        |
| `watch <变量>`       | `watch a`      | 当变量被修改时暂停   |
| `catch throw`      | -              | 捕获 C++ 异常   |
| `shell <命令>`       | `shell ls`     | 执行 shell 命令 |
| `set logging on`   | -              | 将调试输出保存到文件  |

## 九、最常用的调试流程

```bash
# 1. 启动 GDB
gdb ./test.out
# 2. 设置断点
b main
b 12
# 3. 运行程序
r
# 4. 单步执行
n
s
# 5. 查看变量
p a
bt
# 6. 继续运行
c
# 7. 退出
q
```
